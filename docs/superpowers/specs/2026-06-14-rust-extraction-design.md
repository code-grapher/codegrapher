## Symbol model (tree-sitter-rust node kinds → model kinds)
- `mod_item` → `KindModule` (also `KindNamespace`-like; use `KindModule`)
- `struct_item` → `KindStruct`; `enum_item` → `KindEnum`; `enum_variant` → `KindEnumMember`; `union_item` → `KindStruct`
- `trait_item` → `KindInterface` (Rust traits ≈ interfaces)
- `function_item` → `KindFunction` at module scope, `KindMethod` inside an `impl`/`trait` body
- `impl_item` → NOT its own node: its methods attach to the implementing type (qualified `Type::method`); record the (type, trait?) association for resolution
- `const_item`/`static_item` → `KindConstant`
- `field_declaration` (struct field) → `KindField`
- `type_item` (type alias) → `KindTypeAlias`
- `use_declaration` → `KindImport`
- `macro_invocation`/`macro_definition` → skip as nodes this pass (calls inside macro bodies are best-effort)

Flags: `visibility` (`pub`/`pub(crate)`/private), `isAsync` (`async fn`),
`typeParameters` ← generics/lifetimes (names only), `isAbstract` n/a.

## Edges
`contains`, `calls` (`call_expression`; method calls `x.method()` and
`Type::assoc()` — strip `self` receiver), `imports` (`use` path → the imported
item), `extends` n/a for structs; **`implements`** (`impl Trait for Type` →
Type implements Trait), `references` (type usages in fields/params/returns),
`instantiates` (`Type { … }` struct literal and `Type::new(...)` → instantiates
to the struct), `overrides` (a trait method implemented in an `impl Trait for
Type` block → overrides the trait method, mirroring the Go interface-override
synthesis). No `decorates` (Rust attributes `#[derive(...)]` may emit
`references`/`decorates` to the derived trait — best-effort; document).

## Resolution (static, Go-like, hardest)
`resolveRustRef`:
1. **`use` declarations** establish path imports (`use a::b::C;`, `use a::b::*;`,
   `use a::b::{C, D};`, `as` aliases). Build per-file import map + glob set;
   resolve imported names THROUGH to the real cross-file definition.
2. **`impl` association**: methods in `impl Type { … }` attach to `Type`; methods
   in `impl Trait for Type { … }` attach to `Type` AND emit `implements`
   (Type→Trait) + `overrides` (Type::method→Trait::method) edges — a synthesis
   pass analogous to `synthesizeGoImplementsEdges` /
   `synthesizeGoInterfaceOverrideEdges`. Reuse/adapt those if practical.
3. **Names/calls** resolve by name with module preference + through-import;
   `Type::new`/struct-literal → `instantiates`.
4. **Method calls** `x.method()` resolve by method name to the impl method
   (deterministic pick on ambiguity, documented — no full type inference of `x`
   this pass; a struct-literal/`= Type::new()` local binding MAY be inferred like
   Python's constructor inference if cheap and deterministic).
Determinism: build per-file import maps and the impl-association table before
resolving.
