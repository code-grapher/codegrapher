Pure Go, single binary

No cgo, no Node, no WASM runtime. Tree-sitter parsing runs on a pure-Go engine and SQLite via modernc.org/sqlite with FTS5. Builds with CGO_ENABLED=0 and cross-compiles to every target.

Drop-in parity

Same verbs as the original codegraph — init, sync, query, callers, callees, impact, status, files — with byte-for-byte identical JSON, verified against golden outputs.

Library-first

The CLI is a thin consumer. Import the indexer, query, store, resolve, watch, lock and mcp packages straight into your own Go program.

Library-first

Import the graph, not a subprocess.

Open an index, run a ranked query, walk callers and callees — all in-process, with no shell-out and no daemon. The CLI you scripted yesterday is just one consumer of these same packages.

  • Debounced incremental sync via a built-in file watcher.
  • Git-hook integration to sync after merge; works inside git worktrees.
  • PID-file locking so concurrent processes never corrupt the index.
main.go 
package main

import (
    "context"
    "fmt"

    "github.com/specscore/codegrapher/store"
    "github.com/specscore/codegrapher/query"
)

func main() {
    db, _ := store.Open(".codegraph/codegraph.db")
    defer db.Close()

    // Ranked symbol search — FTS5 BM25 with name + path bonuses.
    hits, _ := query.Search(context.Background(), db,
        "Store::Get", query.Opts{Limit: 5})

    for _, h := range hits {
        fmt.Printf("%-24s %s:%d  (%.1f)\n",
            h.Node.QualifiedName, h.Node.FilePath,
            h.Node.StartLine, h.Score)
    }
}

Parity philosophy

Bug-for-bug compatible, on purpose.

CodeGrapher is a behavior-parity port of codegraph (TypeScript, MIT). The golden outputs captured from the original are the spec — so where the original misbehaves, we reproduce it deliberately, because downstream consumers may depend on the observed behavior.

Each intentional quirk carries a TODO(upstream-bug N) marker at the code site and is documented in KNOWN-BUGS.md. Goldens are immutable: every output is re-captured from the original, never hand-edited to make a test pass. That is what makes CodeGrapher a true drop-in replacement and not merely a look-alike.
